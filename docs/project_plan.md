# 1팀 미니 프로젝트 기획서

## 프로젝트 이름

무기 강화 RPG

## 프로젝트 목적

Python을 활용하여 터미널에서 실행되는 종합 게임 프로그램을 만든다.

단순히 게임을 완성하는 것뿐만 아니라, 함수 분리, 파일 분리, GitHub 저장소 사용, 브랜치 작업, Pull Request 또는 merge, 충돌 해결, 팀원 간 역할 분담을 경험하는 것이 목적이다.

## 팀 구성

| 이름 | 역할 |
| --- | --- |
| 최연준 | 팀장, 전체 구조 설계, 메인 메뉴, 저장 기능, 통합, 문서 작성 |
| 최지현 | 무기 강화 기능 |
| 김민기 | 일반 전투 기능 |
| 이동수 | 보스 레이드 기능 |

## 게임 컨셉

플레이어는 하나의 기본 무기를 가지고 시작한다. 무기를 강화하면 더 강한 적을 상대할 수 있다.

게임의 흐름은 다음과 같다.

```text
강화 시도 -> 무기 성장 -> 일반 전투로 골드 획득 -> 다시 강화 -> 보스 레이드 도전
```

세 가지 게임 기능이 서로 독립적으로 보이지 않도록, 모두 같은 플레이어 데이터와 무기 강화 단계, 골드를 공유한다.

## 필수 실행 방식

프로젝트 루트 폴더에서 다음 명령으로 실행한다.

```bash
python main.py
```

실행하면 터미널에 메인 메뉴가 출력되고, 사용자는 번호를 입력하여 기능을 선택한다.

## 메인 메뉴

```text
===== 무기 강화 RPG =====
1. 프로필 보기
2. 무기 강화
3. 일반 전투
4. 보스 레이드
5. 랭킹/기록 보기
6. 저장하고 종료
```

## 주요 기능

### 1. 프로필 보기

현재 플레이어 상태를 보여준다.

- 닉네임
- 현재 골드
- 무기 이름
- 현재 무기 강화 단계
- 최고 강화 단계
- 일반 전투 승패
- 보스 레이드 클리어 단계

### 2. 무기 강화

플레이어가 골드를 사용하여 무기를 강화한다.

규칙:

- 강화 시도에는 골드가 필요하다.
- 강화 단계가 높을수록 강화 비용이 증가한다.
- 강화 단계가 높을수록 성공 확률은 낮아진다.
- 강화 성공 시 무기 강화 단계가 1 증가한다.
- 강화 실패 시 무기 강화 단계가 0으로 초기화된다.
- 최고 강화 단계는 기록으로 남는다.

예시:

| 현재 단계 | 강화 비용 | 성공 확률 |
| --- | ---: | ---: |
| +0 | 100 | 90% |
| +1 | 150 | 80% |
| +2 | 220 | 70% |
| +3 | 300 | 60% |
| +4 | 400 | 50% |
| +5 이상 | 단계별 증가 | 점점 감소 |

### 3. 일반 전투

강화한 무기를 사용하여 랜덤 NPC와 전투한다.

규칙:

- 전투 시작 시 NPC가 랜덤으로 등장한다.
- 플레이어의 전투력은 무기 강화 단계에 따라 증가한다.
- NPC마다 전투력이 다르다.
- 승리하면 골드를 얻는다.
- 패배하면 골드를 조금 잃는다.
- 전투 결과는 기록으로 남는다.

일반 전투는 반복해서 골드를 얻기 위한 콘텐츠다.

### 4. 보스 레이드

정해진 순서대로 보스를 클리어하는 도장깨기 콘텐츠다.

규칙:

- 각 보스는 필요한 무기 강화 단계가 있다.
- 요구 강화 단계보다 낮으면 도전할 수 없다.
- 클리어하면 골드 보상을 받는다.
- 클리어한 최고 스테이지가 저장된다.

예시:

| 스테이지 | 보스 | 필요 강화 단계 | 보상 |
| --- | --- | ---: | ---: |
| 1 | 슬라임 왕 | +0 | 300 |
| 2 | 늑대 대장 | +3 | 600 |
| 3 | 돌 골렘 | +5 | 1000 |
| 4 | 드래곤 | +8 | 2000 |

### 5. 랭킹/기록 보기

현재 저장된 플레이 기록을 보여준다.

- 최고 강화 단계
- 일반 전투 승리 수
- 일반 전투 패배 수
- 보스 레이드 클리어 단계
- 최근 전투 로그
- 최근 레이드 로그

### 6. 저장하고 종료

현재 플레이어 상태를 JSON 파일로 저장하고 프로그램을 종료한다.

## 저장 방식

프로그램을 껐다 켜도 기록이 남도록 JSON 파일을 사용한다.

저장 파일 위치:

```text
data/save_data.json
```

기본 저장 데이터 형태:

```json
{
  "player": {
    "nickname": "player",
    "gold": 1000,
    "weapon_name": "기본 검",
    "weapon_level": 0,
    "best_weapon_level": 0
  },
  "battle": {
    "win": 0,
    "lose": 0,
    "logs": []
  },
  "raid": {
    "cleared_stage": 0,
    "logs": []
  }
}
```

## 협업 방식

각 팀원은 자기 브랜치에서 작업한다.

| 이름 | 브랜치 |
| --- | --- |
| 최연준 | `feature/project-structure` |
| 최지현 | `feature/enhancement` |
| 김민기 | `feature/battle` |
| 이동수 | `feature/raid` |

`main` 브랜치에는 직접 코드를 작성하지 않는다.

작업이 끝나면 GitHub에서 Pull Request를 만들고, 팀장이 확인 후 merge한다.

## 완료 기준

- `python main.py`로 실행된다.
- 메인 메뉴에서 3개 이상의 게임 기능을 선택할 수 있다.
- 무기 강화, 일반 전투, 보스 레이드가 서로 연결된다.
- JSON 저장 기능이 동작한다.
- 팀원별 역할과 작성 파일을 설명할 수 있다.
- 오류 해결 또는 충돌 해결 경험을 회고로 정리한다.
